结合了123题股票Ⅲ 以及 贪心
```
正常情况下，仿照股票Ⅲ，遍历prices，顺序更新purchase[i] 以及 sell[i]， 最后sell[k-1]即为所求。

题中出现k极大的情况，此时动态数组 int *purchase = new int [k] 会溢出。(bad array length)
发现此时与股票Ⅲ不同的是最多进行k次操作。
其实一次操作需要占用两天，即当k=prices.size()/2时，便达到了最大的操作次数。
可以将k赋值为prices.size()/2，但其实这样还是会浪费运算，因为此时已经可以进行最大操作数，即任何两天直接只要股票有升值就可以购买，所以此种情况使用贪心，遍历所有比前一天股票价格高的
prices,累加价格差得到答案。
```
